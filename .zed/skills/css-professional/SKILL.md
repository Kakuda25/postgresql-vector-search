# CSS プロフェッショナル

プロ品質のスタイルを書くための指針。CSS の作成・レビュー・リファクタ時に適用する。

## 基本方針

- **詳細度を抑える**: クラスセレクタを中心にし、ID や過剰なネストで詳細度を上げない。
- **一貫した命名**: プロジェクトで BEM や別の規約を採用している場合はそれに従う。意味が分かる名前をつける。
- **コンテンツに依存しすぎない**: 長さ・語数が変わっても壊れにくいレイアウトと余白にする。

## 詳細度（Specificity）

- スタイルの付与はクラス（`.class`）を基本とする。ID（`#id`）は必要最小限（例: ラベル先頭用）にし、スタイル目的の ID は避ける。
- ネストは 2〜3 段までに抑える。`.block__element` のようにクラスで表現できるならネストで深くしない。
- `!important` はユーティリティや上書きレイヤーなど、意図が明確な場合以外は使わない。詳細度で解決する方を優先する。
- セレクタ例: 避ける `#header nav ul li a.active` → 推奨 `.nav__link.is-active`。

## 命名と構造

- BEM を使う場合: Block（`.block`）、Element（`.block__element`）、Modifier（`.block--modifier` / `.block__element--modifier`）の関係を守る。
- 状態は `.is-active` / `.has-error` などのプレフィックスで表すと、JS との連携が分かりやすい。
- ファイルは「変数・ベース・レイアウト・コンポーネント・ユーティリティ」など役割で分け、プロジェクト方針に合わせる。

## レイアウト

- **モバイルファースト**: ベースは小さい画面用に書き、`min-width` のメディアクエリで段階的に拡張する。
- **Flexbox**: 一方向の並び・均等割り・整列には Flexbox を使う。`flex-grow` / `flex-shrink` / `flex-basis` の意図を明確にする。
- **Grid**: 2次元のレイアウト（カラム・領域割り）には Grid を使う。`grid-template-areas` で構造を読みやすくできる。
- レイアウト用とコンポーネント用の責務を分け、コンテナと子要素の役割をはっきりさせる。

## カスタムプロパティ（変数）

- 色・フォント・余白・ブレークポイントなど、繰り返す値は `--*` のカスタムプロパティにまとめる。
- 名前は役割ベースにする（例: `--color-primary`, `--spacing-unit`）。見た目の値そのものより意味で命名する。
- `:root` やテーマ用クラスで定義し、必要ならメディアクエリ内で上書きする。

## アクセシビリティ

- **フォーカス**: キーボード操作時はフォーカスが分かるようにする。`outline: none` する場合は `:focus-visible` で代替のフォーカススタイルを必ず付ける。
- **色のコントラスト**: テキストと背景のコントラスト比を WCAG に合わせる（AA 以上が目安）。色だけに依存しない情報伝達を心がける。
- **メディア**: `prefers-reduced-motion: reduce` のときはアニメーションを止めるか軽くする。
- クリック・タップ領域は十分なサイズ（目安 44×44px）を確保する。

## パフォーマンス

- セレクタは簡潔にする。`*` や深いネスト、属性セレクタの多用はコストが増える場合がある。
- レイアウトの影響範囲を狭めるため、必要に応じて `contain` を使う（例: `contain: layout`）。
- `will-change` は本当に必要な要素だけに限定し、変更が終わったら外す。乱用しない。
- アニメーションは `transform` と `opacity` を優先する。`width` / `height` / `top` / `left` などはレイアウトやペイントの負荷が大きい。

## レスポンシブ

- ブレークポイントはコンテンツに合わせて決める。デバイス幅だけに縛らず、レイアウトが崩れるポイントで切る。
- フォントサイズや余白は、可能なら `clamp()` や `min()` / `max()` で流動的にする選択肢を検討する。
- 画像・メディアは `max-width: 100%` と `height: auto` で overflow を防ぐ。

## 禁止・非推奨

- インラインスタイルは動的制御が必要な場合以外は避け、クラスで制御する。
- 非推奨プロパティ（例: `-webkit-box` のみに依存した古い Flexbox）は、必要なフォールバック以外使わない。
- 要素型セレクタ（`div`, `span`）だけでスタイルを当てず、クラスと組み合わせるかクラス主体にする。
- 空のルールや未使用のクラスは残さない。

## レビュー時のチェック

- [ ] 詳細度が低く、クラス中心になっているか
- [ ] ネストが深すぎないか、`!important` の乱用はないか
- [ ] 命名が一貫し、役割が分かるか
- [ ] フォーカス表示と `prefers-reduced-motion` を考慮しているか
- [ ] アニメーションに `transform` / `opacity` を優先しているか
- [ ] ブレークポイントとモバイルファーストが妥当か
- [ ] カスタムプロパティで重複を減らしているか
