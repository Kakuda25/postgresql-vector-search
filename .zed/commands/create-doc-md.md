# マークダウンドキュメントの作成

以下に従って、**@order/Docs/** 配下にマークダウンファイル（.md）を作成してください。

## ルール

1. **保存場所**: 必ず `@order/Docs/` の下に作成する
2. **ファイル名**: 日付と内容が分かる形式（例: `2026-02-11_〇〇の調査.md`）。ユーザーがテーマやファイル名を指定した場合はそれを優先する
3. **内容**: ユーザーが「/create-doc-md」の後に続けて書いたテーマ・要件に沿った内容を記述する。指定がなければ「新規ドキュメント」として適切な見出しとプレースホルダを入れる
4. **形式**: マークダウンのみ。コードブロックや表など、可読性の高い構成にする
4. **妥協しない**: - 妥協せず必ず本質的な回答をする。根本原因を見つける。一時的で曖昧な回答は避ける。
## ユーザー入力

ユーザーがスラッシュコマンドの前に書いた文字列（例: 「カメラ機能のWeb/ネイティブ対応一覧」）は、作成するドキュメントのテーマまたはタイトルとして扱ってください。
