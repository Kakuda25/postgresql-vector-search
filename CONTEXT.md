# Vector Search

通販商品カタログを題材に、ベクトル類似検索のデモと運用を行うアプリケーション。UI は利用者と目的が異なる2つの体験を同一アプリ内に持つ。

## Language

**カタログ**:
エンドユーザーが商品を閲覧し、自然言語や商品から「似ている商品」を探すための画面群。
_Avoid_: ショップ, フロント, ストアフロント

**管理画面**:
開発者・運用者が商品マスタの状態確認、Embedding 生成ジョブの実行・監視、類似検索の検証を行う画面群。
_Avoid_: 管理コンソール, バックオフィス, オペレーション画面

**利用者（カタログ）**:
商品を探すエンドユーザー。技術パラメータや運用操作にはアクセスしない。
_Avoid_: 顧客, ショッパー

**利用者（管理画面）**:
Embedding や DB の状態を確認・操作する開発者または運用者。
_Avoid_: 管理者, オペレーター

**カタログエリア**:
エンドユーザー向け画面群。hash ルートは `#/catalog/...` で始まる。ルートは `#/catalog`（トップ）、`#/catalog/search`（類似検索）、`#/catalog/products/:id`（商品詳細）。`#/catalog` ではフィルタ未指定時に全商品をグリッド表示し、カテゴリ選択で絞り込む。独自のナビゲーションを持つ。アプリのデフォルト入口（`/` および `#/`）は `#/catalog` へリダイレクトする。廃止した hash や未知の hash も `#/catalog` へ転送する。通販サイト風の商品閲覧・類似検索を提供するが、購買機能（カート・決済・会員）は含まない。
_Avoid_: フロントエリア, ショップエリア

**商品詳細**:
単一商品の名前・説明・価格・画像等を表示するカタログ画面（`#/catalog/products/:id`）。ページ表示時に基準商品へ `type=product` の類似検索を自動実行し、**類似商品**グリッドを下部に表示する。Embedding 未生成の場合は類似商品を表示せず、準備中メッセージを出す。
_Avoid_: 商品ページ, PDP

**類似商品**:
ベクトル類似検索で得られた、基準商品または検索クエリに意味的に近い商品の一覧。
_Avoid_: おすすめ, 関連商品, レコメンド

**カタログ検索**:
カタログエリアのヘッダー検索バーから実行するベクトル類似検索（`type=text`）。結果は `#/catalog/search` に表示する。検索入力はヘッダーの1箇所のみ。クエリ未指定で `#/catalog/search` に入った場合は空状態メッセージを表示する。`topK`・`scoreThreshold`・類似度スコアは利用者（カタログ）に露出しない。結果の並び順のみが類似度を表す。非表示パラメータのデフォルトは API と同じ（`topK=10`, `scoreThreshold=0`）。商品詳細の類似商品取得も同値を用いる。
_Avoid_: キーワード検索, 商品検索, 全文検索

**カタログデザイン**:
`.cursor/docs/Sample.jpg`（NovaShop モック）を参考にした通販風 UI。左サイドバー（ホーム・カテゴリ一覧）、ヘッダー検索、商品カードグリッド、商品詳細ページを含む。カート・決済・ウィッシュリスト・会員・プロモバナーは含まない。
_Avoid_: 管理画面デザイン, モック完全再現

**店舗名**:
カタログ UI のロゴ・タイトルに表示する架空の通販ブランド名。**Vector Search Store**。プロジェクト名（Vector Search）に由来する。Sample.jpg の「NovaShop」は使用しない。
_Avoid_: NovaShop, Vector Shop, アプリ名, 管理画面

**商品画像**:
商品カード・詳細で表示するビジュアル。`image_url` が設定されていればその画像を表示し、未設定（NULL）の場合はカテゴリ別プレースホルダを表示する。
_Avoid_: サムネイル, 商品写真

**管理エリア**:
運用者向け画面群。hash ルートは `#/admin/...` で始まる。ルートは `#/admin/dashboard`（ダッシュボード）、`#/admin/products`（商品管理）、`#/admin/search`（類似検索検証）、`#/admin/operations`（Embedding 運用）。旧ルート（`#/dashboard` 等）は廃止する。独自のナビゲーションを持つ。レイアウトは表・KPI 中心の機能優先 UI とする。カタログと **デザイントークン**（色・フォント・角丸・ボタン等）を共有する。
_Avoid_: バックオフィスエリア, コンソールエリア

**デザイントークン**:
Sample.jpg を基準とした共通の視覚定数（プライマリカラー、背景色、フォント、角丸、ボタンスタイル等）。カタログ・管理エリアの両方で参照する。
_Avoid_: テーマ, デザインシステム（本プロジェクトでは UI フレームワーク名として使わない）

**エリア間導線**:
カタログエリアと管理エリアを UI 上で行き来するリンク。カタログヘッダーの「管理画面」から `#/admin/dashboard` へ、管理トップバーの「ストアを見る」から `#/catalog` へ遷移する。モック用途の双方向導線。
_Avoid_: ナビゲーション, クロスリンク

## Flagged ambiguities

| 用語 | 旧解釈 | 解決 |
|------|--------|------|
| 通販サイト風 | シードデータのドメイン比喩のみ | カタログ UI としてエンドユーザー向け体験も提供する |

## Example dialogue

> **運用者**: 未生成の Embedding が何件あるか、管理画面のダッシュボードで確認してからジョブを回したい。
>
> **開発者**: 管理画面の `#/admin/operations` で `missing` モードを実行すればいい。カタログ側にはその操作は出さない。
>
> **利用者（カタログ）**: 「軽いランニングシューズ」と入力して、似ている商品を見たい。
>
> **開発者**: ヘッダー検索から **カタログ検索** が走り、`#/catalog/search` に類似商品が並ぶ。商品詳細では基準商品に対する類似商品も見せる。スコアや `topK` は見せない。
>
> **運用者**: `#/admin/dashboard` またはストア右上の「管理画面」から管理エリアに入る。ジョブ完了後は「ストアを見る」で Vector Search Store を確認する。
